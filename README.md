# 나의 기억책 MVP

음성 기반 자서전 생성 서비스의 모바일 웹 MVP입니다.  
이번 단계의 핵심은 자서전 읽기 화면(Viewer)을 **고령층도 편하게 읽고 들을 수 있는 전자책형 화면**으로 개선하는 것입니다.

## 프로젝트 상태

- React + TypeScript + Vite 기반 모바일 웹앱
- 실제 녹음 파일을 STT와 Gemini로 처리해 자서전 생성
- 시작 화면, 녹음 화면, 생성 진행 화면, 보관함, 자서전 읽기 화면 구현
- 실제 STT/API 연동 없이 MVP 시연용 더미 흐름 제공

## 현재 Viewer 기능

- 표지
- 목차
- 본문 페이지
- 이전/다음 페이지 이동
- 좌우 스와이프
- 현재 페이지 번호
- TTS 낭독/정지
- 글자 크기 조절
- 마지막 읽은 위치 저장

## Viewer 개선 목표

읽기 화면은 앱 화면처럼 보이기보다, 사용자가 생성된 자서전을 **책처럼 읽고 음성으로 듣는 경험**에 집중해야 합니다.  
특히 50~80대 사용자를 기준으로 정보량을 줄이고, 본문 가독성과 하단 조작 편의성을 우선합니다.

## Viewer 화면 구조도

```txt
자서전 읽기 화면
├─ 본문 영역
│  ├─ 표지
│  ├─ 목차
│  └─ 본문 페이지
│
├─ 최소 정보 표시
│  └─ 작은 페이지 번호
│
└─ 하단 조작 영역
   ├─ 이전
   ├─ 읽기 / 정지
   ├─ 다음
   └─ 글자 크기
```

## 제거할 요소

- 본문보다 눈에 띄는 큰 상단 제목
- 앱 이름을 반복해서 보여주는 상단 바
- 강한 카드형 테두리와 과한 그림자
- 읽기 중 계속 노출되는 불필요한 상태 표시
- 복잡한 버튼 그룹 배치

## 축소할 요소

- 페이지 번호는 작게 표시
- 상단 UI는 필요할 때만 최소 노출
- 목차와 본문 헤더의 시각적 무게 축소
- 버튼 텍스트와 영역은 단순화
- 책 페이지 그림자는 약하게 조정

## 새로 추가할 요소

- 몰입 모드
  - 일정 시간 터치가 없으면 상단/하단 UI 자동 숨김
  - 화면 터치 시 UI 다시 표시

- 하단 중심 조작
  - 이전, 다음, TTS 버튼을 엄지손가락이 닿기 쉬운 하단에 배치
  - 자주 쓰는 기능을 한 줄로 정리

- 고령층 기준 본문 스타일
  - 기본 글자 크기 확대
  - 줄간격과 문단 간격 확대
  - 좌우 여백 안정화
  - 한 화면에 너무 많은 문장이 들어가지 않도록 조정

## UX 개선 이유

- 본문이 가장 먼저 보이면 사용자가 읽기에 바로 집중할 수 있습니다.
- 상단 정보를 줄이면 앱 조작 화면이 아니라 전자책을 읽는 느낌이 강해집니다.
- 하단 조작은 안드로이드 스마트폰 한 손 사용에 더 적합합니다.
- 자동 숨김 UI는 읽는 중 시야 방해를 줄입니다.
- 큰 글자, 넓은 줄간격, 충분한 문단 간격은 고령층의 피로도를 낮춥니다.
- TTS 버튼을 하단에 명확히 배치하면 "읽기"와 "듣기" 전환이 쉬워집니다.

## React 컴포넌트 수정 제안

```txt
components/reader/
├─ BiographyReader.tsx
│  ├─ 몰입 모드 상태 관리
│  ├─ 터치 시 UI 표시
│  ├─ 페이지 이동 시 TTS 정지 유지
│  └─ 하단 조작 영역 중심으로 재배치
│
├─ BookPage.tsx
│  ├─ 본문 여백 확대
│  ├─ 문단 간격 개선
│  └─ 장/절 제목 시각적 무게 축소
│
├─ PageNavigation.tsx
│  ├─ 하단 고정 배치
│  └─ 이전/다음 버튼 단순화
│
├─ TTSControl.tsx
│  ├─ 읽기/정지 버튼을 하단 주요 버튼으로 배치
│  └─ 현재 페이지 낭독 유지
│
└─ FontSizeControl.tsx
   ├─ A-/A+ 유지
   └─ 하단 보조 조작으로 배치
```

## 유지해야 할 기능

- 표지, 목차, 본문 페이지
- 이전/다음 이동
- 좌우 스와이프
- TTS 낭독/정지
- 페이지 이동 시 TTS 자동 정지
- 글자 크기 조절
- 읽은 위치 저장
- localStorage 기반 글자 크기 저장

## 실행 방법

```bash
npm install
npm run dev
```

```txt
http://localhost:5173
```

## 빌드 확인

```bash
npm run build
```

현재 빌드 성공 확인 완료.

## 발표 요약

현재 MVP는 자서전 읽기 기능이 동작하는 상태입니다.  
다음 개선 방향은 기능을 늘리는 것이 아니라, 읽기 화면에서 본문을 중심에 두고 상단 정보를 줄이며, 하단 조작과 몰입 모드를 통해 고령층도 편하게 읽고 들을 수 있는 전자책형 Viewer로 다듬는 것입니다.
